What It Answers
 

A sequence is one unit's ordered categorical states over time — a member's monthly account status, a patient's care stages, an employee's roles.

yasqat takes these in long format — the shape event tables already have — and answers three questions: how similar are two sequences, what types exist in a population, and how do states flow over time.

That is the whole scope. No event calculus, no plotting — those stay in your existing stack.

Where yasqat sits.

01 · R · SINCE 2008
TraMineR
The field's reference implementation — the full method family, wide-format (STS) matrices, plotting included. yasqat treats it as the correctness oracle: expected values in our tests are pinned to TraMineR's output.
the oracle
02 · PYTHON · INRIA
TanaT
Temporal sequence analysis focused on patient care pathways, with a broader temporal model — events, intervals and states in multi-sequence trajectories — and built-in visualisations. Our reference for Python API conventions.
the neighbor
03 · PYTHON · POLARS
yasqat
Deliberately narrower: categorical state sequences only, long-format-first, no plotting layer. Built to sit inside an existing data platform rather than beside it — DataFrames in, DataFrames out.

Differences
04 / 10
vs. TraMineR and TanaT — in practice

Four deliberate differences.

long
Canonical input is one row per (id, time, state) — the shape event tables already have.
TraMineR's wide STS matrix is a view you can derive, not a prerequisite you must build.
compiled
polars frames end to end; the O(n²) pairwise distance kernels are numba-compiled
over integer-encoded arrays. No pandas in the core.
pinned
Test expectations are TraMineR's numbers, not re-derived approximations.
When in doubt, match R.
tables
Cluster labels (id · cluster · is_medoid), indicators, rules — results are polars DataFrames
that join straight back onto the tables they came from.
The fifth difference is scope itself: state sequences only — events and intervals stay in TanaT's territory.

Round Trip
 
# Spark side — export the long-format event table
events.write.parquet("s3://lake/trajectories/")

# Python side — load, map columns, analyse
df = pl.read_parquet("s3://lake/trajectories/")
pool = load_dataframe(df, config=SequenceConfig(
    id_column="user_id", time_column="ts", state_column="event"))

dm = pool.compute_distances(method="om")
labels = pam_clustering(dm, n_clusters=5).to_dataframe()

labels.write_parquet("s3://lake/trajectory_clusters/")

SequenceConfig maps your column names; pass an explicit Alphabet and unexpected states fail at load time instead of becoming silent categories.

After the Clusters
07 / 10
A typology is not a one-off study

Choose, test, operate, rehearse.

— 01 / CHOOSE

Pick k with evidence

pam_range sweeps candidate cluster counts and reports silhouette and quality indices for each — the number of types is chosen, not assumed.

— 02 / TEST

Permutation-tested

discrepancy_analysis computes pseudo-R² and pseudo-F with a permutation p-value — does tenure, region or cohort actually explain trajectory differences?

— 03 / OPERATE

Fit once, score later

PAM's predict() assigns new sequences to the nearest medoid from their distances to the training set — the typology becomes a scoring step, not a one-off analysis.

— 04 / REHEARSE

Synthetic first

generate_financial_journeys produces synthetic banking journeys — onboarding, product adoption, risk events — so the pipeline is built and demoed before production data is touched.

Scope
09 / 10
In the box · out of the box

What it does — and doesn't.

IN shipping today
Does

Distances: the Optimal Matching family, Hamming, LCS/LCP, DTW, TWED. Clustering: PAM, CLARA, hierarchical, with quality indices. Statistics: descriptive and normative indicators, discrepancy analysis, subsequence mining and association rules. Synthetic generators for prototyping.

  • Expected values pinned against TraMineR
  • Every result a polars DataFrame
OUT by design, or not yet
Doesn't

No plotting — results are tables; bring matplotlib, altair or your BI layer. No event or interval calculus — categorical states only. And it is pre-1.0: breaking changes still land in minor versions, documented in the changelog.

  • Backlog in the open — richer subsequence mining, distance-engine internals
  • 1.0 = a committed stable interface
