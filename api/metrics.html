Metrics

Pairwise distance functions for sequence comparison. All functions accept integer-encoded numpy arrays and return a scalar distance. Metric classes implement compute() (one pair) and pairwise() (full pool).

from yasqat.metrics import DistanceMatrix, SequenceMetric
from yasqat.metrics.optimal_matching import OptimalMatchingMetric, optimal_matching_distance
from yasqat.metrics.hamming import HammingMetric, hamming_distance

DistanceMatrix

@dataclass
class DistanceMatrix(values, labels=None)

Wrapper for a symmetric pairwise distance matrix.

Parameters

Name Type Description
values np.ndarray Square symmetric matrix of shape (n, n).
labels list[int \| str] \| None Sequence IDs corresponding to rows/columns.

Methods

Method Returns Description
to_condensed() np.ndarray Upper-triangle condensed form for scipy.
from_condensed(condensed, labels) DistanceMatrix Class method — build from condensed form.

Example

dist = pool.compute_distances("om", indel_cost=1.0)
print(dist.values.shape)   # (n, n)
print(dist.labels[:5])     # sequence IDs

SequenceMetric

class SequenceMetric(ABC)

Abstract base class for all metric classes. Subclass this to implement custom metrics.

Abstract methods

def compute(self, seq_a: np.ndarray, seq_b: np.ndarray, **kwargs) -> float: ...
def pairwise(self, pool: SequencePool) -> DistanceMatrix: ...

Custom metric example

from yasqat.metrics.base import SequenceMetric, DistanceMatrix
import numpy as np

class MyMetric(SequenceMetric):
    name = "my_metric"

    def compute(self, seq_a, seq_b, **kwargs):
        return float(np.sum(seq_a != seq_b))

    def pairwise(self, pool):
        ids = pool.sequence_ids
        seqs = [pool[i].get_encoded() for i in range(len(pool))]
        n = len(seqs)
        mat = np.zeros((n, n))
        for i in range(n):
            for j in range(i + 1, n):
                d = self.compute(seqs[i], seqs[j])
                mat[i, j] = mat[j, i] = d
        return DistanceMatrix(values=mat, labels=ids)

build_substitution_matrix

def build_substitution_matrix(
    n_states,
    method="constant",
    cost=2.0,
    transition_rates=None,
    state_frequencies=None,
) -> np.ndarray

Build a substitution cost matrix for use with Optimal Matching.

method Description
"constant" All substitutions cost cost (default 2.0).
"trate" Costs derived from empirical transition rates.
"indels" 1/freq(a) + 1/freq(b) — requires state_frequencies.
"indelslog" Log variant of "indels".
"future" Chi-squared distance between next-state distributions.
"features" Gower distance on user-defined state feature vectors.

Metric reference

Optimal Matching (OM)

def optimal_matching_distance(
    seq_a: np.ndarray,
    seq_b: np.ndarray,
    indel_cost: float = 1.0,
    substitution_costs: np.ndarray | None = None,
) -> float

Edit distance with custom insertion/deletion and substitution costs. The gold-standard metric for categorical sequence analysis (Abbott & Forrest 1986).

class OptimalMatchingMetric(indel_cost=1.0, substitution_costs=None)
    def compute(seq_a, seq_b) -> float
    def pairwise(pool) -> DistanceMatrix

Example

from yasqat.metrics.optimal_matching import OptimalMatchingMetric
from yasqat.statistics import substitution_cost_matrix

sub = substitution_cost_matrix(pool)          # data-driven costs
metric = OptimalMatchingMetric(substitution_costs=sub, indel_cost=1.0)
dist = metric.pairwise(pool)

def lcs_distance(seq_a: np.ndarray, seq_b: np.ndarray) -> float

1 − LCS_length / max(len_a, len_b). Invariant to insertions/deletions.

Soft-DTW

def softdtw_distance(

    seq_b: np.ndarray,
    gamma: float = 1.0,
) -> float

Differentiable relaxation of DTW (Cuturi & Blondel 2017). Useful as a smooth training loss.

Time-Warp Edit Distance (TWED)

def twed_distance(
    seq_a: np.ndarray,
    seq_b: np.ndarray,
    nu: float = 0.001,
    lmbda: float = 1.0,

Combines edit distance with time-warping; nu controls stiffness, lmbda is the deletion cost.

Chi-squared (Chi2)

def chi2_distance(seq_a: np.ndarray, seq_b: np.ndarray) -> float

Chi-squared distance between the state frequency distributions of two sequences. Order-invariant — treats sequences as bags of states.

Discrepancy-based Hamming Distance (DHD)

def dhd_distance(seq_a: np.ndarray, seq_b: np.ndarray) -> float

Weighted Hamming distance accounting for state-level discrepancy.

Euclidean

def euclidean_distance(seq_a: np.ndarray, seq_b: np.ndarray) -> float

Standard L2 norm between integer-encoded sequences.

OM Variants

def omloc_distance(seq_a, seq_b, indel_cost=1.0) -> float
def omspell_distance(seq_a, seq_b, indel_cost=1.0) -> float
def omstran_distance(seq_a, seq_b, indel_cost=1.0) -> float

OM extensions (Halpin 2010) that penalise state changes differently:

  • OMLoc — position-sensitive OM; earlier mismatches cost more.
  • OMSpell — spell-duration-sensitive OM.
  • OMStran — penalises transition-heavy sequences more strongly.

NMS family

def nms_distance(seq_a: np.ndarray, seq_b: np.ndarray) -> float
def nmsmst_distance(seq_a: np.ndarray, seq_b: np.ndarray) -> float
def svrspell_distance(seq_a: np.ndarray, seq_b: np.ndarray) -> float

Non-metric spell-based distances (Studer & Ritschard 2016):

  • NMS — number of matching subsequences.
  • NMSMst — NMS on the most frequent subsequence.
  • SVRSpell — Spell variant.
